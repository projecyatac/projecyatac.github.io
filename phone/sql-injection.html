Описание

 

Инъекции SQL - атака состоит из вставки или «инъекции» запрос SQL через входные данные от клиента к приложению. Успешный SQL-инъекционный эксплойт может считывать конфиденциальные данные из базы данных, изменять данные базы данных (вставка / обновление / удаление), выполнять операции администрирования базы данных (например, отключение СУБД), восстанавливать содержимое данного файла, присутствующего в файле СУБД система, а в некоторых случаях выдает команды операционной системе. Атаки с использованием SQL-инъекций представляют собой тип атаки с использованием инъекций , в которой команды SQL вводятся во входные данные плоскости данных для выполнения предопределенных команд SQL.

 

Ошибки внедрения SQL возникают, когда:

 

Данные поступают в программу из ненадежного источника.

Данные, используемые для динамического построения запроса SQL

 

Основными последствиями являются:

 

Конфиденциальность . Поскольку базы данных SQL обычно содержат конфиденциальные данные, потеря конфиденциальности является частой проблемой уязвимостей SQL-инъекций .

Аутентификация . Если для проверки имен пользователей и паролей используются плохие команды SQL, возможно, можно подключиться к системе как другой пользователь, ранее не знавший пароль.

Авторизация . Если информация об авторизации хранится в базе данных SQL, возможно, можно изменить эту информацию за счет успешного использования уязвимости SQL-инъекции .

Целостность . Точно так же, как можно прочитать конфиденциальную информацию, можно также внести изменения или даже удалить эту информацию с помощью атаки SQL-инъекцией .
