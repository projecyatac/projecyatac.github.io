Описание

 

Бинарная установка - это общий термин для атаки, когда злоумышленник помещает (то есть, устанавливает) двоичный файл, содержащий вредоносный код, в локальную или удаленную файловую систему, чтобы уязвимое приложение могло загрузить и выполнить его.

 

Эта атака может происходить разными способами:

 

Небезопасные права доступа к локальному каталогу позволяют локальному злоумышленнику разместить вредоносный двоичный файл в надежном расположении. (Типичным примером является установщик приложения, который неправильно настраивает разрешения для каталогов, используемых для хранения файлов приложения.)

Одно приложение может использоваться для установки вредоносного двоичного файла в надежном расположении другого приложения.

Приложение ищет двоичный файл в ненадежных местах, возможно, в удаленных файловых системах. (Типичным примером является приложение Windows, загружающее библиотеку динамических ссылок из текущего рабочего каталога после того, как последний был установлен в общую сетевую папку.)

 

Примеры

Атака на основе небезопасного доступа

Установщик приложений Windows создает корневой каталог (C: \ Application) и устанавливает в него приложение, но не может ограничить доступ для записи в каталог для непривилегированных пользователей.

Предположим, что приложение (C: \ Application \ App.exe) загружает библиотеку WININET.DLL, вызывая LoadLibrary ("WININET.DLL"). Предполагается, что эта библиотека находится в папке Windows System32.

Локальный пользователь A устанавливает вредоносную библиотеку WININET.DLL в C: \ Application

Локальный пользователь B запускает приложение, которое загружает и запускает вредоносный файл WININET.DLL вместо легитимного.

Текущая рабочая атака на основе Directroy

Предположим, что приложение Windows загружает библиотеку DWMAPI.DLL, вызывая LoadLibrary ("DWMAPI.DLL"). Предполагается, что эта библиотека находится в папке Windows System32, но существует только в Windows Vista и Windows 7.

Предположим, что приложение связано с расширением .bp.

Злоумышленник устанавливает сетевую общую папку и помещает файлы honeypot.bp и DWMAPI.DLL в эту папку (возможно, помечая последнюю как скрытую).

Злоумышленник приглашает пользователя Windows XP посетить общую папку с помощью проводника Windows.

Когда пользователь дважды щелкает по honeypot.bp, проводник Windows устанавливает текущий рабочий каталог для удаленного общего ресурса и запускает приложение для открытия файла.

Приложение пытается загрузить DWMAPI.DLL, но, не найдя его в системных каталогах Windows, загружает и запускает его из общей сетевой папки злоумышленника.
